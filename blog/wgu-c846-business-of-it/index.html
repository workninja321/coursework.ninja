WGU C846 Business of IT: Project+ Certification Guide

January 2, 2026 8 min read Course Help

WGU C846 Business of IT – Project Management prepares you for the CompTIA Project+ (PK0-005) certification. This vendor-neutral credential validates your ability to manage small to medium-sized projects, making it valuable for IT professionals moving into coordination or management roles. Pass the exam, complete the course, and add a recognized certification to your resume.

For more WGU certification guidance, see our WGU Certification Guide. For the full IT degree picture, check the WGU BSIT Complete Guide.

Exam overview

The CompTIA Project+ PK0-005 exam tests your knowledge of project management fundamentals across the entire project lifecycle.

Exam details:

  • Exam code: PK0-005
  • Questions: Maximum of 95 questions
  • Question types: Multiple choice and performance-based
  • Time: 90 minutes
  • Passing score: 710 out of 900
  • Validity: Does not expire (unlike PMP)

Exam domains and weights

CompTIA organizes the exam into four domains. Focus your study time according to these weights:

Domain Weight Description
1.0 Project Basics 36% Project characteristics, constraints, roles, communication, change management
2.0 Project Constraints 17% Schedule, budget, scope, quality, resources, risk
3.0 Communication and Change Management 26% Stakeholder engagement, meetings, documentation, change control
4.0 Project Tools and Documentation 21% Charts, diagrams, project management software, reports
Pro tip: Domain 1 (Project Basics) accounts for over a third of the exam. Master this domain thoroughly before moving on.

Domain 1: Project Basics (36%)

This is the largest domain and covers foundational project management concepts.

Project characteristics

  • Temporary: Projects have defined start and end dates
  • Unique: Each project produces a distinct deliverable
  • Progressive elaboration: Details become clearer as project progresses

Project vs. operational work

Projects are temporary and create unique outcomes. Operations are ongoing and repetitive. Know the difference for exam questions.

Key roles

  • Project sponsor: Provides funding and high-level direction
  • Project manager: Plans, executes, and closes the project
  • Stakeholders: Anyone affected by or interested in the project
  • Project team: People doing the work
  • PMO: Project Management Office, provides governance and standards

Project lifecycle phases

  1. Initiation: Define scope, stakeholders, create project charter
  2. Planning: Develop schedule, budget, risk plan, communication plan
  3. Execution: Perform the work, manage team, acquire resources
  4. Monitoring and Controlling: Track progress, manage changes
  5. Closing: Finalize deliverables, lessons learned, release resources

Domain 2: Project Constraints (17%)

Understand the triple constraint and how project managers balance competing demands.

The triple constraint

  • Scope: What the project will deliver
  • Schedule: When it will be delivered
  • Cost: How much it will cost

Changes to one constraint affect the others. If scope increases, schedule or cost (or both) must also increase.

Additional constraints

  • Quality: Standards the deliverables must meet
  • Resources: People, equipment, materials available
  • Risk: Uncertain events that could impact the project

Domain 3: Communication and Change Management (26%)

This domain covers stakeholder management and how to handle changes during a project.

Communication methods

  • Formal written: Project plans, contracts, status reports
  • Informal written: Email, instant messages, notes
  • Formal verbal: Presentations, scheduled meetings
  • Informal verbal: Conversations, impromptu discussions

Stakeholder engagement

  • Identify all stakeholders early
  • Assess their interest and influence
  • Create a communication plan tailored to each group
  • Manage expectations throughout the project

Change control process

  1. Receive change request
  2. Log and document the request
  3. Analyze impact (scope, schedule, cost, risk)
  4. Submit to change control board (CCB)
  5. Implement approved changes
  6. Update documentation
Common exam trap: Changes should never be implemented without going through the formal change control process, even if they seem minor.

Domain 4: Project Tools and Documentation (21%)

Know the common tools project managers use to plan, track, and communicate project status.

Planning tools

  • Work Breakdown Structure (WBS): Hierarchical decomposition of deliverables
  • Gantt chart: Visual timeline showing tasks and dependencies
  • Network diagram: Shows task dependencies (Critical Path Method)
  • PERT chart: Estimates duration using optimistic, pessimistic, and most likely

Key documents

  • Project charter: Authorizes the project, names the PM
  • Scope statement: Defines what is (and isn't) included
  • Risk register: Tracks identified risks and responses
  • RACI matrix: Responsible, Accountable, Consulted, Informed
  • Lessons learned: Captured at project close

4-week study plan

  1. Week 1: Domain 1 – Project Basics (36% of exam)
  2. Week 2: Domain 3 – Communication and Change Management (26%)
  3. Week 3: Domain 4 – Tools and Documentation (21%) + Domain 2 – Constraints (17%)
  4. Week 4: Practice exams, review weak areas, exam day

Exam day strategies

  • Skip performance-based questions first: Flag them and return after multiple choice
  • Read carefully: Watch for "BEST" or "MOST" – there may be multiple correct answers
  • Eliminate wrong answers: Narrow down before guessing
  • Think like a project manager: Choose the systematic, process-oriented answer
  • Manage your time: 95 questions in 90 minutes = less than 1 minute per question

Frequently Asked Questions

C846 prepares you for the CompTIA Project+ (PK0-005) certification exam. Passing the exam completes the course and earns you an industry-recognized project management certification.

Project+ is considered an entry-level project management certification. Most students with some project experience find it manageable with 2-4 weeks of focused study. The exam tests conceptual knowledge rather than hands-on skills.

Plan for 2-4 weeks of study. Students with prior project management experience often pass in 1-2 weeks, while those new to the field should budget the full 4 weeks to thoroughly understand all domains.

Project+ demonstrates foundational project management skills valued in IT roles. While not as prestigious as PMP, it's vendor-neutral and recognized across industries. It's particularly valuable for IT professionals transitioning into management or coordination roles.
