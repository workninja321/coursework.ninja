WGU C170 Data Management: SQL and Database Fundamentals Guide

December 21, 2025 8 min read Course Help

WGU C170 Data Management (often labeled Data Management - Applications) is the SQL-heavy course that builds your database foundation. The WGU Institutional Catalog highlights MySQL, SQL syntax, database and table creation, views, indexes, and queries ranging from simple single-table statements to complex multi-table analysis.

Course focus (official WGU catalog)

  • Use SQL queries to analyze and extract data
  • Build working knowledge of the MySQL database engine
  • Create and modify databases and tables
  • Work with views and indexes
  • Write SQL queries from single-table to multi-table joins
Prerequisite: The WGU catalog lists Data Management - Foundations as a prerequisite to Data Management - Applications.

SQL skills you must be able to execute

Expect to spend most of your time in SQL. Build a habit of writing queries from scratch, not just reading them.

  • SELECT with WHERE, ORDER BY, and LIMIT
  • Aggregations with GROUP BY and HAVING
  • INNER, LEFT, and RIGHT joins across multiple tables
  • Subqueries for filtering and comparison
  • DDL: CREATE, ALTER, and DROP
  • DML: INSERT, UPDATE, and DELETE
  • Constraints, keys, and indexes

Data modeling and normalization

SQL alone is not enough. You also need to design clean relational structures. Focus on:

  • Entities, attributes, and relationships
  • Primary and foreign keys
  • Cardinality and optionality in ER diagrams
  • Normalization to reduce redundancy (1NF through 3NF)
Common pitfall: Students often skip normalization checks and lose points on design-based tasks.

MySQL workflow you should practice

The catalog specifically calls out MySQL, so practice in a MySQL environment. Make sure you can:

  • Create databases and tables from scratch
  • Define keys and constraints correctly
  • Create views for reusable queries
  • Add or modify indexes for performance

Study plan (2 to 4 weeks)

  1. Week 1: SQL basics, SELECT queries, and single-table practice.
  2. Week 2: Joins, aggregations, and subqueries with real datasets.
  3. Week 3: Data modeling, normalization, and schema design practice.
  4. Week 4 (optional): Mixed practice sets and focused review of weak areas.

How to practice SQL efficiently

  1. Build a small schema with 3 to 5 related tables.
  2. Write questions first, then write the SQL that answers them.
  3. Check results for edge cases (NULLs, duplicates, missing relationships).
  4. Refactor queries to be shorter and clearer.

Assessment approach

Expect to demonstrate both conceptual understanding and hands-on SQL ability. A good rule: every requirement should map to a query, a table design decision, or a clear explanation.

Common mistakes to avoid

  • Memorizing syntax without building real queries
  • Skipping ER diagram practice
  • Not validating query output against requirements
  • Confusing joins or using the wrong key
