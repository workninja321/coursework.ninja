WGU C949 Data Structures and Algorithms I: Complete Study Guide

January 2, 2025 12 min read Course Help

WGU C949 (Data Structures and Algorithms I) is a foundational course in the Computer Science program that teaches you how to organize data efficiently and solve problems with optimal algorithms. If you can implement data structures from scratch, analyze time complexity, and choose the right tool for each problem, you're ready to pass. This guide covers every major topic, practical study strategies, and the mindset you need to succeed.

Course overview

C949 covers the fundamental building blocks of computer science: how to store, organize, and manipulate data efficiently. You'll learn both the theory (why certain structures work) and the practice (how to implement them in code).

Key topics include:

  • Linear data structures: arrays, linked lists, stacks, queues
  • Non-linear data structures: trees, graphs, heaps
  • Hash tables and hashing functions
  • Sorting algorithms: bubble, selection, insertion, merge, quick
  • Searching algorithms: linear search, binary search, tree traversals
  • Algorithm analysis and Big O notation
  • Recursion and its applications

Big O notation: The foundation

Before diving into specific data structures, you must understand Big O notation. This mathematical notation describes how an algorithm's performance scales with input size.

Common time complexities (fastest to slowest):

Notation Name Example
O(1) Constant Array index access
O(log n) Logarithmic Binary search
O(n) Linear Linear search
O(n log n) Linearithmic Merge sort, quick sort
O(n^2) Quadratic Bubble sort, selection sort
O(2^n) Exponential Recursive Fibonacci
Pro tip: When analyzing algorithms, focus on the worst-case scenario and drop constants and lower-order terms. O(2n + 5) simplifies to O(n).

Arrays and dynamic arrays

Arrays are the simplest data structure: a contiguous block of memory storing elements of the same type.

Operations and complexity:

  • Access by index: O(1)
  • Search (unsorted): O(n)
  • Insert at end: O(1) amortized for dynamic arrays
  • Insert at beginning/middle: O(n)
  • Delete: O(n)

When to use arrays:

  • You need fast random access by index
  • The size is known or changes infrequently
  • You're implementing other data structures (heaps, hash tables)

Linked lists

A linked list stores elements in nodes, where each node contains data and a pointer to the next node. This allows efficient insertions and deletions without shifting elements.

Types:

  • Singly linked: Each node points to the next
  • Doubly linked: Each node points to both next and previous
  • Circular: The last node points back to the first

Operations and complexity:

  • Access by index: O(n)
  • Insert at head: O(1)
  • Insert at tail (with tail pointer): O(1)
  • Insert in middle: O(n) to find position, O(1) to insert
  • Delete: O(n) to find, O(1) to remove

Stacks and queues

Stacks and queues are abstract data types that restrict how you add and remove elements.

Stacks (LIFO - Last In, First Out)

  • Push: Add to top - O(1)
  • Pop: Remove from top - O(1)
  • Peek: View top without removing - O(1)

Use cases: Function call stack, undo operations, expression evaluation, backtracking algorithms

Queues (FIFO - First In, First Out)

  • Enqueue: Add to back - O(1)
  • Dequeue: Remove from front - O(1)
  • Peek: View front without removing - O(1)

Use cases: BFS traversal, task scheduling, print queues, message buffers

Trees

Trees are hierarchical structures with a root node and child nodes. They're fundamental for organizing data with parent-child relationships.

Binary trees

Each node has at most two children (left and right).

Binary search trees (BST)

A binary tree where left children are smaller and right children are larger than the parent.

  • Search: O(log n) average, O(n) worst case (unbalanced)
  • Insert: O(log n) average, O(n) worst case
  • Delete: O(log n) average, O(n) worst case

Tree traversals

  • In-order (LNR): Left, Node, Right - produces sorted output for BST
  • Pre-order (NLR): Node, Left, Right - useful for copying trees
  • Post-order (LRN): Left, Right, Node - useful for deleting trees
  • Level-order: Breadth-first traversal using a queue
Common mistake: Confusing traversal orders. Use the mnemonic: In-order is "in between" (L-N-R), Pre-order visits node "first" (N-L-R), Post-order visits node "last" (L-R-N).

Heaps

A heap is a complete binary tree that satisfies the heap property. Used to implement priority queues.

  • Max-heap: Parent is greater than or equal to children
  • Min-heap: Parent is less than or equal to children

Operations:

  • Insert: O(log n)
  • Extract max/min: O(log n)
  • Get max/min: O(1)
  • Build heap: O(n)

Hash tables

Hash tables provide near-constant time operations by using a hash function to compute an index for each key.

Operations (average case):

  • Insert: O(1)
  • Search: O(1)
  • Delete: O(1)

Collision handling:

  • Chaining: Store colliding elements in a linked list at each index
  • Open addressing: Find the next available slot (linear probing, quadratic probing)

Graphs

Graphs model relationships between objects. Vertices (nodes) are connected by edges.

Representations:

  • Adjacency matrix: 2D array, good for dense graphs, O(1) edge lookup
  • Adjacency list: Array of lists, good for sparse graphs, O(V) space

Traversals:

  • Depth-First Search (DFS): Uses a stack, explores as deep as possible first
  • Breadth-First Search (BFS): Uses a queue, explores level by level

Sorting algorithms

Simple sorts (O(n^2))

  • Bubble sort: Compare adjacent pairs and swap if out of order
  • Selection sort: Find minimum and move to front
  • Insertion sort: Build sorted portion by inserting elements one by one

Efficient sorts (O(n log n))

  • Merge sort: Divide and conquer, stable, requires O(n) extra space
  • Quick sort: Divide and conquer using a pivot, in-place, unstable
  • Heap sort: Build a heap and extract elements, in-place, unstable
Algorithm Best Average Worst Space Stable
Bubble O(n) O(n^2) O(n^2) O(1) Yes
Selection O(n^2) O(n^2) O(n^2) O(1) No
Insertion O(n) O(n^2) O(n^2) O(1) Yes
Merge O(n log n) O(n log n) O(n log n) O(n) Yes
Quick O(n log n) O(n log n) O(n^2) O(log n) No

Searching algorithms

  • Linear search: O(n) - check each element sequentially
  • Binary search: O(log n) - requires sorted array, divide search space in half
// Binary search pseudocode
function binarySearch(arr, target):
    left = 0
    right = arr.length - 1
    
    while left <= right:
        mid = (left + right) / 2
        if arr[mid] == target:
            return mid
        else if arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    
    return -1  // not found

Recursion

Recursion solves problems by breaking them into smaller subproblems of the same type. Every recursive function needs:

  • Base case: A condition that stops the recursion
  • Recursive case: The function calls itself with a smaller input

Common recursive patterns:

  • Tree traversals
  • Merge sort and quick sort
  • Fibonacci sequence
  • Factorial calculation
  • Binary search

Study strategy

  1. Understand before memorizing: Know WHY each data structure works, not just HOW
  2. Implement from scratch: Write each data structure without looking at references
  3. Visualize: Draw structures and trace through algorithms by hand
  4. Practice problems: Use LeetCode, HackerRank, or similar platforms
  5. Analyze complexity: For every algorithm you write, determine its Big O

4-week study plan

  1. Week 1: Big O notation, arrays, linked lists, stacks, queues
  2. Week 2: Trees (binary trees, BST), tree traversals, heaps
  3. Week 3: Hash tables, graphs, DFS, BFS
  4. Week 4: Sorting algorithms, searching, recursion, comprehensive review

Assessment tips

  • Know the Big O complexity of every operation on every data structure
  • Be able to trace through algorithms by hand (show your work)
  • Understand when to use each data structure for a given problem
  • Practice explaining your reasoning - many questions test understanding, not just recall

Recommended resources

  • Visualgo.net: Visualizations of data structures and algorithms
  • LeetCode Easy problems: Start with array and linked list problems
  • YouTube - Abdul Bari: Excellent algorithm explanations
  • GeeksforGeeks: Comprehensive reference with implementations

Frequently Asked Questions

C949 covers fundamental data structures (arrays, linked lists, stacks, queues, trees, graphs, hash tables) and algorithms (sorting, searching, traversal). Big O notation and algorithm analysis are also key components.

Most students complete C949 in 4-8 weeks depending on their programming background. Those with prior algorithm experience may finish faster, while beginners should plan for the full 8 weeks.

C949 is considered one of the more challenging courses in the CS program because it requires both conceptual understanding and practical implementation skills. Success depends on consistent practice and visualization of data structures.

Strong programming fundamentals in at least one language and completion of Discrete Mathematics I are recommended. Familiarity with recursion is particularly helpful.
